# DZ3 Unity

## Краткое объяснение по двери и анимациям

### Почему `Trigger`
`Trigger` подходит для взаимодействия (вход в зону, нажатие `E`) без физического толчка объекта. Для двери это удобно: мы проверяем условие "можно взаимодействовать" и запускаем анимацию.

### Структура переходов в Animator
Для двери:
- `Closed` (по умолчанию)
- `Open`
- переход `Closed -> Open` по параметру `DoorTrigger` (тип `Trigger`)

Для игрока:
- `Idle` (default), `Run`, `Jump`
- параметры: `Speed` (Float), `Jump` (Bool)
- переходы:
  - `Idle -> Run` при `Speed > 0.1`
  - `Run -> Idle` при `Speed <= 0.1`
  - `Any State -> Jump` при `Jump == true`
  - `Jump -> Idle` при `Jump == false` и `Speed <= 0.1`
  - `Jump -> Run` при `Jump == false` и `Speed > 0.1`

### Логика `cooldown`
После успешного нажатия `E` скрипт сохраняет время следующего допустимого взаимодействия:
`nextInteractTime = Time.time + interactionCooldown`.
Пока текущее время меньше этого значения, повторные нажатия игнорируются. Это защищает от спама триггера и дёрганых повторных запусков анимации.

### Когда `Trigger`, а когда `Collision`
- `Trigger`: для логики взаимодействия (зона активации двери, детекция входа).
- `Collision`: для физического контакта (упор в дверь, блокировка прохода, реакции физики).

Практический вариант для двери:
- Trigger-коллайдер вокруг двери для проверки взаимодействия.
- Обычный collider у створки двери для физической геометрии.
